# Kronos-TH

A research project that runs walk-forward backtests using the Kronos foundation model to forecast Thai equity returns and generate daily trading signals.

## Language

**Backtest**:
A historical simulation that replays what would have happened if the model's trading signals were executed live, using strict walk-forward methodology (no look-ahead bias). Produces a `BacktestResult` containing equity curve, trades, and metrics.
_Avoid_: Simulation, replay (too generic)

**Expected Return**:
The raw numerical forecast output from the Kronos model: `(predicted P50 price / current price) − 1`. Computed per ticker per day by `compute_signals()` in `kth/backtest/strategy.py`. Used for ranking stocks within an asset class.
_Avoid_: Raw signal, forecast score

**Trade Signal**:
The categorical action derived from Expected Return after applying thresholds: BUY (Expected Return > 2.0%), HOLD (1.0%–2.0%), SELL (<1.0%). Displayed in daily reports as the "Signal" column.
_Avoid_: Entry/exit signal (too vague)

**Ranked Signal**:
The confidence-adjusted final recommendation, computed by `build_report_table()`. Combines Trade Signal with backtest-calibrated confidence flags (green/yellow/red). Displayed with the "Score" column in the daily brief.
_Avoid_: Final signal, adjusted signal

**Trade**:
A single execution event: buying or selling shares of a ticker at a specific price on a specific date. Stored as one row in `trades.parquet`. Has a direction (BUY/SELL), size (THB or % of portfolio), and gross return (for closed legs).
_Avoid_: Transaction, execution (too generic)

**Position**:
The accumulated shares of a single ticker across multiple trades. Computed as: `sum(all buys) - sum(all sells)`. The backtest engine tracks this as `holdings_units` in `walkforward.py`. A position can span days or weeks.
_Avoid_: Holding, allocation (allocation is the target weight, not the actual shares)

**Portfolio Value**:
The mark-to-market (MTM) total of all positions plus cash, evaluated at each day's closing prices. Tracked as `equity_curve` in `BacktestResult`. Used to compute daily returns, drawdowns, and Sharpe.
_Avoid_: Account value, NAV, equity (too overloaded)

**CAGR Alpha**:
The simple difference in annualized returns: `CAGR_strategy - CAGR_benchmark`. Used in period decomposition tables and trader-oriented reports. Intuitive: "we beat the benchmark by 23 percentage points per year."
_Avoid_: Alpha (use the prefix to disambiguate)

**OLS Alpha**:
The regression intercept × 252 from a linear regression of strategy returns against benchmark returns. Measures excess return after removing market beta exposure. Used in statistical tables and quant-oriented reports. Computed by `compute_metrics()` in `kth/backtest/metrics.py`.
_Avoid_: Alpha, Jensen's alpha (Kronos uses equal-weight benchmark, not CAPM market portfolio)

**Forecast**:
The Kronos model's prediction: an expected future price (P50) for a given ticker over a given horizon (5d, 20d). Generated by `forecast_batch()` and stored in `ForecastResult` objects. Can be persisted to parquet (for backtests) or returned directly (for the daily brief). "Precompute" is a legacy term for batch-forecasting to disk — all predictions are forecasts.
_Avoid_: Prediction, precompute (implementation detail, not a domain concept)

**Entry Threshold**:
The minimum Expected Return required for a stock to qualify as a BUY signal. Currently 2.0% (`long_threshold + entry_buffer = 1.0% + 0.5%`). A stock must clear this bar to enter the top-5 portfolio.
_Avoid_: Buffer, upper threshold

**Hold Threshold**:
The minimum Expected Return required for an existing Position to remain in the portfolio. Currently 1.0% (`long_threshold`). A stock whose Expected Return falls below this is sold. This creates hysteresis: exiting is easier than entering, preventing portfolio churn.
_Avoid_: Lower threshold, exit threshold

**Risk Band**:
A dynamic position-sizing rule that adjusts capital deployment based on trailing performance. Three bands: BULL (15% allocation, 12-week Sharpe > 1.0), NEUTRAL (10%, Sharpe 0.5–1.0), BEAR (5%, Sharpe 0.0–0.5), EXIT (0%, Sharpe < 0). Checked weekly (Friday) and executed the next trading day.
_Avoid_: Allocation band, position size rule

**Circuit Breaker**:
A hard safety cut-off that liquidates all Positions when the Portfolio Value drops 10% from its all-time peak. Overrides the Risk Band. Re-enters when the Portfolio Value recovers 3% above the stop level. Implemented as the `−10% trail stop + 3% re-entry` rule.
_Avoid_: Stop-loss (too generic, implies a per-position stop), emergency exit

**Equal-Weight Benchmark**:
A passive portfolio that holds all 50 Thai equity tickers at equal weight, rebalanced daily. Serves as the canonical benchmark for measuring stock-selection alpha: beating EW means the model's picks outperform the broader universe. Used in `compute_metrics()` for OLS Alpha and statistical tests.
_Avoid_: Benchmark (use the full name to disambiguate from other benchmarks)

**SET Benchmark**:
A buy-and-hold of the SET Index (`^SET.BK`). Used for narrative context — "the model beat the market by 37 percentage points" — not for measuring alpha. The SET is a cap-weighted index that doesn't control for universe composition the way Equal-Weight does.
_Avoid_: Market benchmark, Thai benchmark

**Forecast Horizon**:
The number of trading days ahead that the model predicts. Currently two horizons: 5-day (short-term tactical) and 20-day (medium-term signal used for ranking). The model forecasts the expected closing price at each horizon for every ticker.
_Avoid_: Prediction length, look-ahead window

**P50**:
The median (50th percentile) of the model's predicted price distribution for a given ticker at a given Forecast Horizon. Interpreted as "the model's best guess." Used to compute Expected Return: `(P50 / current_price) − 1`. The model also outputs P5 and P95 for confidence bands.
_Avoid_: Median prediction, expected price

**Information Ratio (IR)**:
A measure of stock-selection skill: `annualised active return / tracking error vs Equal-Weight Benchmark`. Higher IR means more return per unit of deviation from the benchmark. Used in the Signal Health row alongside Sharpe. Computed by `compute_information_ratio()` in `kth/backtest/metrics.py` (planned Phase 3).
_Avoid_: Alpha ratio, active Sharpe (imprecise)

**Batting Average**:
The percentage of calendar months where the strategy's mean daily return exceeded the Equal-Weight Benchmark's mean daily return. Measures consistency, not magnitude. A strategy can have high Sharpe but low batting average (a few big wins). Computed by `compute_batting_average()` (planned Phase 3).
_Avoid_: Win rate (that's trade_win_rate — per-trade P&L, not monthly benchmark-beating)

**Calibration**:
A check on whether the model's P5/P95 forecast bands are statistically honest. Compares each historical forecast's [P5, P95] range to the actual price 20 trading days later. Target: actual prices fall inside the band ~90% of the time. Coverage < 80% = overconfident; > 95% = overly wide. Computed by `compute_calibration()` (planned Phase 3) and shown in the Signal Health row.
_Avoid_: Band accuracy, confidence accuracy

**Drawdown Velocity**:
A regime warning triggered when the Portfolio Value drops more than 3% over 5 consecutive trading days, even without hitting a single-day threshold. Catches slow grinds that the Circuit Breaker misses. Displayed as an 8th tile in the Risk Bar when triggered. Computed by `compute_drawdown_velocity()` (planned Phase 4).
_Avoid_: Grind (too informal), slow drawdown

**Bootstrap p-value**:
A statistical measure of whether the strategy's live alpha is real or luck. Computed by shuffling the strategy's daily returns 1,000 times and counting what fraction of shuffles beat the Equal-Weight Benchmark by at least the observed margin. p < 0.05 = edge confirmed; p ≥ 0.15 = no confirmed edge. Displayed inline in the Signal Health row. Computed by `compute_bootstrap_pvalue()` (planned Phase 4).
_Avoid_: p-value (use full term to distinguish from other statistical tests)

**Sector Concentration**:
A portfolio-level risk flag triggered when the buy loop would place more than 2 positions in the same SET sector (e.g., Banking, Energy, Property). Enforced by a hard filter in `trade_gen.py` using the `SECTOR` dict in `universe.py`. Not a warning — it silently skips over-concentrated picks and continues to the next ranked ticker.
_Avoid_: Sector cap, concentration limit

**Survivorship Bias**:
An upward distortion in backtest CAGR caused by the universe containing only currently-listed tickers. Delisted Thai stocks (bankruptcies, mergers, 2020–2024) are excluded because yfinance does not expose their historical data. Estimated to inflate CAGR by ~1–3 percentage points per year for 40–60 stock universes. Formal disclosure added to `docs/backtest-methodology.html` (planned Phase 4).
_Avoid_: Data bias, selection bias (too generic)

## Example Dialogue

**Trader:** "What does today's report say?"  
**Dev:** "The model's **P50** for CENTEL at the **20-day Forecast Horizon** is 33.80, current price is 30.50, so the **Expected Return** is +10.8%. That clears the **Entry Threshold** of 2.0%, so CENTEL gets a BUY **Trade Signal**. It's ranked #1, so it's in the **Position** at 20% weight."  
**Trader:** "What about risk?"  
**Dev:** "The **Risk Band** is NEUTRAL — 12-week Sharpe is 0.82, between 0.5 and 1.0, so we're at 10% allocation. The **Circuit Breaker** is not triggered — **Portfolio Value** drawdown is −5.3%, still room before the −10% cut."  
**Trader:** "How do we know this works?"  
**Dev:** "The latest **Backtest** shows **CAGR Alpha** of +23.3% over **Equal-Weight Benchmark** — the model's stock picks consistently outperform just holding all 50 stocks equally. The SET was down 15% that same year, so the **SET Benchmark** confirms we're not just riding a bull market."  
**Trader:** "What's the difference between **Expected Return** and **Trade Signal**?"  
**Dev:** "Expected Return is the raw number the model outputs — 'this stock should go up 6.8% in 20 days.' Trade Signal is the thresholded version — 'this clears 2.0%, so BUY. That one only clears 1.5%, so HOLD.' The **Ranked Signal** adds confidence calibration from the backtest — 'BUY, but the model's confidence on this ticker class is only 3%.'"  
**Trader:** "What happens if the Portfolio Value drops 12%?"  
**Dev:** "The **Circuit Breaker** triggers. All **Positions** are sold, the portfolio goes to cash. We wait for recovery to 3% above the stop level before re-entering. During this time, the **Risk Band** is overridden — even if Sharpe is great, we stay out."
