## Start the Flask dashboard (one command)
```bash
./scripts/start_dashboard.sh
```
Idempotent: creates venv, installs both requirements files, downloads data (skips if any parquet < 24h old), runs the forecast pipeline, serves on port 5555, waits for `/api/health`. Subcommands: `stop`, `restart`, `status`, `logs`, `clean`. Env vars: `KRONOS_PORT` (default 5555), `INITIAL_CAPITAL`, `KRONOS_MODE` (default paper). Run the pipeline in the **evening** so tomorrow's forecast uses today's close.

## Google Suite dashboard — architecture an agent might miss
- **Kaggle (primary daily runtime)** — scheduled notebook (`kaggle/kronos_kaggle_pipeline.ipynb`, generated by `build_kaggle_notebook.py`), runs each evening BKK time. GPU + Internet + Secrets. Writes to same Google Sheets as Colab. Idempotent by design (date-keyed upserts for Equity Curve + Forecast History).
- **Colab (backup runtime)** = daily compute engine (notebook generated by `build_notebook.py`, "Run All" each morning).
- **Google Sheets** = persistent data store + fill-confirmation input surface.
- **Apps Script web app** = 5-tab dashboard SPA (`apps_script/Code.gs` backend, `apps_script/Index.html` frontend).
- **JSON-bridge pattern:** Sheets → read fills → Drive JSON → existing `kth` functions → Drive JSON → Sheets. `os.chdir(KTH_REPO)` in Cell 1 makes `Path("data/...")` resolve against Drive. Cell 9 (update portfolio state) must run before Cell 10 (generate ticket).
- `migrate_to_sheets.py` ports `data/positions/paper_portfolio.json` + `trade_log.csv` to Sheets (fresh-start only). Setup: `docs/SETUP_GUIDE.md`.

## Key conventions an agent might miss

### Kronos schema (enforced in `kth/data/loader.py`)
- Columns must be exactly: `timestamps, open, high, low, close, volume, amount`.
- `amount = close * volume` (Yahoo does not expose turnover).
- yfinance returns `Open/High/Low/Close/Volume` with a DatetimeIndex; `to_kronos_format()` lowercases and renames.

### Caching
- One **parquet per ticker**, never a merged file (different date ranges per asset class). Default dir `./data/raw/`.
- Filename sanitization: `^` → `_`, `=` → `_` (e.g. `^SET.BK` → `_SET.BK`, `THB=X` → `THB_X`).
- `auto_adjust=True` so splits/dividends are baked in.
- Gaps are **preserved**, not forward-filled (crypto 7 days vs equities 5).
- `download_universe()` pauses 0.5s between tickers, retries with exponential backoff (2s/4s/8s).

### Universe & asset classes
- Hardcoded in `kth/data/universe.py` (100 tickers, 9 classes) — not a CSV by design.
- `FRICTION` is per-class, not per-ticker. `fx_macro` is **features only**, not investable (zero friction).
- `SECTOR` dict + `get_sector()` map 50 thai_equity tickers to 10 SET sectors for the sector guard (max 2 positions/sector).

## Research decisions that constrain code (do not relitigate without a GPU re-run)

Full numbers live in `data/backtest_results/` and `PROJECT_STRUCTURE.md`. The durable conclusions:

- **Position sizing: equal-weight only.** `inv_vol` was backtested and lost badly — it over-allocates to low-vol stocks where the Kronos signal is weakest. Do NOT switch to inv_vol.
- **Fine-tuning: everything stays zero-shot.** FT did not beat ZS in backtest for thai_equity, us_equity, or crypto. (Direction-accuracy gains did not translate to backtest alpha.)
- **Alpha is regime-conditional, not year-round.** Strategy strongly beats flat/down SET regimes (2024/2025) but underperforms in broad bull markets (2023, and 2026-to-date). In a bull regime, reduce to BEAR allocation until the regime shifts or paper trades accumulate.
- **Factor-neutral:** OLS vs SET market + 12-1 momentum gives beta ≈ 0 and residual alpha ≈ +29%/yr — genuine model alpha, not factor exposure or a momentum proxy.
- **Statistical caveat:** only the 2024 OOS year clears p<0.05 (single test); under Bonferroni no year survives. Treat the edge as regime-conditional.
- **min_holding_days / long_threshold experiments are invalid** — they read the general n10 forecast cache while stored results use n50 precomputed forecasts. Do not change parameters based on them without GPU re-precompute.
- **Bootstrap p-value distinction — do not confuse:** `compute_bootstrap_pvalue()` (live dashboard `/api/risk`, centered resampling) vs historical backtest p-values (t-test in `compute_metrics()`, stored in `data/backtest_results/`, never changed).
- **Kronos pre-training cutoff ≈ Dec 2022 (inferred).** 2023-2026 OOS backtests are clean; the 2022 portion of older runs may be partially in-sample. Confirm via arXiv:2508.02739 §3 if it matters.
- **Metrics naming:** `hit_rate` was renamed `trade_win_rate` (trade P&L, not forecast direction accuracy). Crypto uses a 7-day calendar (`date_range(D/B)`, not `bdate_range`).
- **GULF.BK** excluded from forecasts until ~Oct 2026 (only 281 rows, needs ≥400); auto-qualifies once history accumulates.
